# Internationalization (i18n) Plan

This document explains the multilingual system for the Mafia Game.

---

## Architecture

```
[Backend] → Emits events with eventKey + params
            ↓
[Client] → Receives event → Renders localized string from ARB file
```

**Key Principle:** Backend NEVER sends user-facing text. Only event keys and parameters.

---

## Event Key Pattern

### Format

```
category.action[.result]
```

### Examples

```
auth.registered
auth.login_failed
crime.success
crime.failed
crime.caught
job.completed
hospital.healed
vehicle.purchased
crew.created
crew.disbanded
bank.deposit_success
travel.arrived
aviation.flight_started
```

## Backend Event Emission

### Structure

All backend responses follow this pattern:

```typescript
{
  "event": "crime.success",
  "params": {
    "crimeName": "pickpocket",
    "moneyEarned": 500,
    "xpGained": 10,
    "location": "downtown"
  },
  "timestamp": "2026-01-27T12:34:56Z"
}
```

### Implementation

`backend/src/services/worldEventService.ts`:

```typescript
export interface GameEvent {
  eventKey: string;
  params: Record<string, any>;
  playerId?: string;
  isPublic: boolean;
}

export function createEvent(event: GameEvent) {
  return prisma.worldEvent.create({
    data: {
      eventKey: event.eventKey,
      params: JSON.stringify(event.params),
      playerId: event.playerId,
      isPublic: event.isPublic,
    }
  });
}
```

### Event Emitter Helper

`backend/src/utils/eventEmitter.ts`:

```typescript
export function emitEvent(
  eventKey: string,
  params: Record<string, any> = {},
  playerId?: string,
  isPublic: boolean = false
) {
  const event = {
    event: eventKey,
    params,
    timestamp: new Date().toISOString(),
  };

  // Store in database
  createEvent({
    eventKey,
    params,
    playerId,
    isPublic,
  });

  // Broadcast via SSE/WebSocket if public
  if (isPublic) {
    eventBroadcaster.broadcast(event);
  }

  return event;
}
```

## Client Event Renderer

`client/lib/services/event_renderer.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:flutter_gen/gen_l10n/app_localizations.dart';

class EventRenderer {
  final AppLocalizations l10n;

  EventRenderer(this.l10n);

  String render(String eventKey, Map<String, dynamic> params) {
    // Convert eventKey from dot notation to underscore
    // e.g., "crime.success" → "crime_success"
    final key = eventKey.replaceAll('.', '_');

    switch (key) {
      case 'auth_registered':
        return l10n.auth_registered(params['username'] ?? '');
      
      case 'auth_login_failed':
        return l10n.auth_login_failed;
      
      case 'crime_success':
        return l10n.crime_success(
          params['crimeName'] ?? '',
          params['moneyEarned'] ?? 0,
          params['xpGained'] ?? 0,
        );
      
      case 'crime_failed':
        return l10n.crime_failed(
          params['crimeName'] ?? '',
          params['moneyLost'] ?? 0,
        );
      
      case 'crime_caught':
        return l10n.crime_caught(
          params['crimeName'] ?? '',
          params['bailAmount'] ?? 0,
        );
      
      case 'job_completed':
        return l10n.job_completed(
          params['jobName'] ?? '',
          params['earnings'] ?? 0,
        );
      
      case 'hospital_healed':
        return l10n.hospital_healed(
          params['healthRestored'] ?? 0,
          params['cost'] ?? 0,
        );
      
      case 'vehicle_purchased':
        return l10n.vehicle_purchased(
          params['vehicleType'] ?? '',
          params['price'] ?? 0,
        );
      
      case 'crew_created':
        return l10n.crew_created(
          params['leaderName'] ?? '',
          params['crewName'] ?? '',
        );
      
      case 'crew_disbanded':
        return l10n.crew_disbanded(params['crewName'] ?? '');
      
      case 'bank_deposit_success':
        return l10n.bank_deposit_success(params['amount'] ?? 0);
      
      case 'travel_arrived':
        return l10n.travel_arrived(
          params['country'] ?? '',
          params['cost'] ?? 0,
        );
      
      case 'aviation_flight_started':
        return l10n.aviation_flight_started(
          params['playerName'] ?? '',
          params['origin'] ?? '',
          params['destination'] ?? '',
          params['aircraftType'] ?? '',
        );
      
      // Fallback: return raw event key if not found
      default:
        return 'Unknown event: $eventKey';
    }
  }
}
```

### Usage in Widget

```dart
class EventFeedWidget extends StatelessWidget {
  final List<GameEvent> events;

  const EventFeedWidget({required this.events});

  @override
  Widget build(BuildContext context) {
    final l10n = AppLocalizations.of(context)!;
    final renderer = EventRenderer(l10n);

    return ListView.builder(
      itemCount: events.length,
      itemBuilder: (context, index) {
        final event = events[index];
        final message = renderer.render(event.eventKey, event.params);
        
        return ListTile(
          title: Text(message),
          subtitle: Text(formatTimestamp(event.timestamp)),
        );
      },
    );
  }
}
```

## Translation Workflow

### 1. Developer Adds New Event

When adding a new feature, developer:

1. Define event key in backend (e.g., `"heist.success"`)
2. Emit event with params in service code
3. Add translations to both ARB files
4. Update `EventRenderer` switch case
5. Test in both languages

### 2. Translator Updates Strings

Non-technical translators can:

1. Open `app_nl.arb` (or other language)
2. Edit translations (JSON values only)
3. Do NOT change keys or placeholders
4. Save and commit

### 3. Adding New Language

To add, e.g., Spanish:

1. Create `client/lib/l10n/app_es.arb`
2. Copy structure from `app_en.arb`
3. Translate all values
4. Update `client/l10n.yaml`:

```yaml
arb-dir: lib/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
supported-locales:
  - en
  - nl
  - es
```

5. Run `flutter gen-l10n`
6. Add `es` to `MaterialApp.supportedLocales`

      child: Column(
        children: [
          Text(name, style: Theme.of(context).textTheme.headline6),
          Text(description),
        ],
      ),
    );
  }
  
  String _getTranslation(AppLocalizations l10n, String key) {
    // Use a map or code generation to access translations by key
    // For simplicity, you can use a switch or a map
    final translations = {
      'crime_name_pickpocket': l10n.crime_name_pickpocket,
      'crime_desc_pickpocket': l10n.crime_desc_pickpocket,
      // ... add all keys
    };
    return translations[key] ?? key;
  }

## Best Practices

### ✅ DO

- Always use eventKey + params pattern
- Keep params simple (strings, numbers, booleans)
- Use snake_case for event keys
- Test in all supported languages before releasing
- Keep ARB files in sync (same keys in all languages)
- Use descriptive event keys (e.g., `crime.caught` not `crime.fail2`)

### ❌ DON'T

- Send user-facing text from backend
- Hardcode strings in client code
- Mix languages in same string
- Use HTML or markdown in ARB files (unless necessary)
- Change placeholder types between languages
- Forget to update EventRenderer when adding new event

## Testing i18n

### Manual Test

```dart
void main() {
  final enLocale = AppLocalizations.delegate.load(Locale('en'));
  final nlLocale = AppLocalizations.delegate.load(Locale('nl'));
  
  final event = {
    'eventKey': 'crime.success',
    'params': {'crimeName': 'pickpocket', 'moneyEarned': 500, 'xpGained': 10}
  };
  
  final enRenderer = EventRenderer(enLocale);
  final nlRenderer = EventRenderer(nlLocale);
  
  print(enRenderer.render(event['eventKey'], event['params']));
  // "You successfully committed pickpocket and earned $500! (+10 XP)"
  
  print(nlRenderer.render(event['eventKey'], event['params']));
  // "Je hebt succesvol pickpocket gepleegd en $500 verdiend! (+10 XP)"
}
```

## Future Expansion

### Additional Languages

To add more languages:
1. Create `app_<locale>.arb`
2. Translate all strings
3. Add to `l10n.yaml`
4. Test

### Pluralization

Use ICU message format for plurals:

```json
{
  "players_online": "{count, plural, =0{No players} =1{1 player} other{{count} players}} online",
  "@players_online": {
    "placeholders": {
      "count": {"type": "int"}
    }
  }
}
```

### Gender Support

If needed, use `select` in ARB:

```json
{
  "welcome_message": "{gender, select, male{Welcome, Mr. {name}} female{Welcome, Ms. {name}} other{Welcome, {name}}}",
  "@welcome_message": {
    "placeholders": {
      "gender": {"type": "String"},
      "name": {"type": "String"}
    }
  }
}
```

## Summary

✅ Backend emits `eventKey` + `params`  
✅ Client renders localized strings from ARB files  
✅ Translators only edit ARB values (not code)  
✅ Content names also use translation keys  
✅ System supports unlimited languages with minimal code changes  

This architecture keeps backend and client decoupled, allows non-technical translators to work independently, and makes adding languages trivial.
